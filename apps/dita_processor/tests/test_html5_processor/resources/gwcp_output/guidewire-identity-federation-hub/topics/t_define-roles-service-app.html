Specify roles for service applications

The roles returned in the SAML response identify the permissions for the request.

For service applications, you must choose the roles that specify the permissions for the service. When Guidewire Hub receives a validation request from your application, it determines whether the requested scope is valid. Guidewire Hub then returns the valid role a scope.
  1. Define the roles that you will use. If necessary, create new roles.

    The suggested naming convention is to name the role after your service. For example, for a service named mycompany_allstarrepairintegration, you would name the role mycompany_allstarrepairintegration.

  2. Create the YAML files that assign the permissions to the roles.

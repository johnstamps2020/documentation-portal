Choose a data access level

Your application can have different levels of data access depending on who the users are. An application can support one or more of the following access levels:
Access level Description
cc_gwabuid External users whose resource access is determined by whether they are associated with an assigned specialist on a service request.
cc_policyNumbers External users whose resource access is determined by the policies to which they have rights.
cc_username Internal users who are stored in ClaimCenter.

Although an application may support multiple access levels, each call to the system API must specify exactly one access level.

Ensure that your IdP tracks the information necessary to support the chosen access levels.

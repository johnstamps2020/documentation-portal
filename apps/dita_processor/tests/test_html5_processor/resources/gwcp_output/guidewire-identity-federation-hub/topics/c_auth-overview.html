Guidewire authentication and authorization

Guidewire must control access to its applications and data. When a user tries to access a Guidewire resource, the user must be authenticated, and then verified to be authorized to access that resource. Authentication and authorization (together often called auth) is a common requirement for cloud applications, and there are many products available that provide these services.

The Guidewire Identity Federation Hub (Guidewire Hub) is an auth server that uses OAuth 2.0 and SAML for identity management services. Guidewire Hub is the trusted auth server for all Guidewire cloud applications, including client applications that you create that access Guidewire cloud resources.

Guidewire Hub uses your identify provider (IdP) for user management. When a user logs in to your custom application, the application makes an auth request to Guidewire Hub, which then federates the request to your IdP.

To enable Guidewire Hub to use your IdP, you must have your IdP registered with the Guidewire Hub. In addition, if you want to use your own client application to access Guidewire resources, you must register your application.

Register your identity provider (IdP)

Registering your IdP with Guidewire allows your organization to use Guidewire Hub for auth functions.

Registration allows Guidewire Hub to federate auth requests to your IdP. You need to register your IdP only once.
  1. Ensure that you have an operational SAML-based IdP.
  2. Contact your Guidewire Services representative to register your IdP.
Guidewire will provide you with the following information. Store this information for later use.
idp_id
The ID of your IdP. You will use this ID when issuing requests to the Guidewire Hub.
tenant_id
The ID of your organization. You will use this ID when registering Guidewire applications with Guidewire Hub.

Register your InsuranceSuite applications

Registering allows your Guidewire InsuranceSuite applications to use Guidewire Hub for auth functions.

You will likely use multiple environments for your Guidewire application. For example, you might have a staging environment and a production environment. You must register each application for each environment separately.

  1. Contact your Guidewire Services representative to register your application.
  2. Guidewire will provide you with the following information. Store this information for later use.
    authServerUri
    The URI for Guidewire Hub.
  3. Modify the the SignatureKeyProviderPlugin plugin.
    Note: This step is likely to change in a future release.
    1. In Guidewire Studio, navigate to configuration > config > Plugins > registry, and then open the SignatureKeyProviderPlugin.gwp plugin.
    2. In the Parameters list, for the parameter defaultIssuer, set the Value to the value of authServerUri provided to you by Guidewire.
    3. In the Parameters list, for the parameter allowedIssuers, set the Value to the value of authServerUri provided to you by Guidewire.
  4. Modify the RestAuthenticationVerifierPlugin plugin.
    Note: This step is likely to change in a future release.
    1. In Guidewire Studio, navigate to configuration > config > Plugins > registry, and then open the RestAuthenticationVerifierPlugin.gwp plugin.
    2. In the Parameters list, for the parameter tenantId, set the Value to the value of tenant_id provided to you by Guidewire.
  5. Change the universal variable in testbench to use the tenant ID provided to you by Guidewire.

Managing secrets

With secrets, you can store sensitive data, for example authentication credentials to gain access to external systems.

Note: To work with secrets, you must use the latest version of the SecretManagerCredentialsPlugin plugin. Contact your customer partner.
A secret is a key-value pair, where:
  • Key – Secret identifier. A key must consist only of lower case letters and periods. Periods can only be used between letters. For example: credentials.to.external.system
  • Value – Sensitive data. For authentication data, the format must be: user:password. For example: JohnSmith:Password!123

To use a secret, refer to its key in your code.

Once you add a secret, you can edit its value.

After you add a secret, deploy the related applications to see the changes.

Secret context

When adding a secret, you can specify the context to which the secret applies. The context can be a project, an environment or an application. For example, you can add a secret for all applications deployed in a selected environment.

In each context, you can assign different values to a key. To do this, add a number of secrets with the same key and specify the context for each secret. During runtime, key values are imported to applications in order from the most specific to the least specific assignment.

For example, you can create two secrets with the key some.key and specify the following values: user:passwordCC for ClaimCenter and user:passwordAll for all applications in the environment. During deployment, the key some.key returns the value user:passwordCC for ClaimCenter and user:passwordAll for any other application deployed in the specified environment.

Credentials plugin

When you add authentication credentials in Cloud Console and refer to them in the application code, they are passed to the SecretManagerCredentialsPlugin plugin. Cloud Console passes the data in the following environment variables:
  • PLUGIN_AWS_REGION
  • PLUGIN_AWS_SECRET_NAMES
This process is automatic.
Note: Use the latest version of the credentials plugin. Contact your customer partner.

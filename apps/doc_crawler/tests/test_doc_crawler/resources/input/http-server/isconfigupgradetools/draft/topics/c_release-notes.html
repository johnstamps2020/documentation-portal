Release notes

This topic describes changes in InsuranceSuite Configuration Upgrade Tools 4.1.0.

Improvements

ID Description
PSUPGPOD-7245 This implementation includes two improvements:
  • Upgrade steps that were a part of Services Upgrade Tools are now part of Upgrade Tools and follow their implementation standards. Migrated steps have the services-repo prefix. We have also added ADMIN_DATA fileSet to support document transformations when running the upgrade steps.
  • We added missing .dti mappings to the ScriptParametersEnhancement step.
PSUPGPOD-7460 To allow user-added JARs to appear in Merge Tracker, only idea-gclasses and gclasses are excluded from Merge Tracker, with the rest of the plugin folder remaining intact.
PSUPGPOD-6365 Upgrading entities that included a typekey reference with a default set to "" (blank string) caused an error. Currently, in such a situation, the default attribute is removed.
PSUPGPOD-7395 To help you with your upgrade, we have introduced the following filter groups to Merge Tracker:
  • Aggregate Type
  • Conflicts
  • Other Filters
  • For details on new filters, see Available Merge Tracker filters.
  • For information on how to edit filters, see Edit a Guidewire filter.
PSUPGPOD-7127, PSUPGPOD-6975 We have added the following new upgrade steps to improve the upgrade process:
  • 8.0.0 upgrade steps
  • 8.0.1 upgrade steps
  • Replace DocumentContents (10.0.0)

  • add-value-type-to-product-model.step.yaml

  • replace-f-k-owner.step.yaml

Upgrade steps reference
PSUPGPOD-7063 During an upgrade, primitive and boxed types such as int, short, and long no longer result in a reduction of precision when placed on the right side of a comparison expression.
PSUPGPOD-7066 When comparing GX models, different line endings would appear as differences. To fix that, we added .gx as a text file extension in the Merge Tracker settings.
PSUPGPOD-7059 We fixed an issue where XML files containing character references that could not be represented by their specified encoding were truncated at the position of the illegal character during the XML reformatting step. Now, the XML reformatting step converts such files to UTF-8 and the character references are converted to individual characters.
PSUPGPOD-7056 File names containing special characters no longer cause the details of the file exported from Merge Tracker to be invalid. Now exporting file details from Merge Tracker produces valid and correct output even when file names include special characters such as commas and quotes.
PSUPGPOD-7045 Even if a config/inbound-integration.xml file has nested XML comments, the upgrade process now upgrades the file correctly instead of failing.
PSUPGPOD-7044

The ScriptParametersEnhancementUpgradeStep creates a file Ex_ScriptParametersEnhancement.gsx which contains virtual properties for each script parameter defined in ScriptParameters.xml. This produced a compilation error when upgrading from versions higher or equal to 9.

To fix this problem, Ex_ScriptParametersEnhancement.gsx is created only when the source InsuranceSuite version is lower than 9.0.0 and the target InsuranceSuite version is higher than or equal to 9.0.0.

PSUPGPOD-7043 Merge Tracker File comparison window title now contains the relative path to the opened file.

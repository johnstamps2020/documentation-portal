PolicyCenter 2022.05.1 Release Notes

New and changed

Free-text search in InsuranceSuite for Guidewire Cloud

Availability of free-text search in InsuranceSuite for Guidewire Cloud is dependent on support in Guidewire Cloud Platform.

Before configuring or using free-text search, contact your Guidewire representative or operating partner to verify that free-text search is supported.

Hooks for Rewrite and Reinstatement Process now available

Additional hooks for the Rewrite and Reinstatement Process are now available through the runPreBindAndIssue and runPostBindAndIssue methods. Additionally, you can call the new setPolicyOriginalEffectiveDate protected method from the SubmissionProcessImpl.finishBinding method.

Improvements and resolved issues

Guidewire attempts to provide information for issues of primary importance to our customers. This list is not intended to be comprehensive.

PolicyCenter improvements and resolved issues

ID Description
ISPC‑3523 Error messages for High Volume Quoting (HVQ) and quote searches no longer prompt you to try again later.

Advanced Product Designer improvements and resolved issues

ID Description
APD‑2403 When adding new options to a previously code-generated clause term using APD, the lookup entries were not added for the new options, causing the terms to be unavailable or the server to fail to start. This issue has been resolved. The lookups are now generated correctly when a new option is added to a term.
APD‑2408 When deleting an edition in APD (cloud UI) this operation would not be reflected in any PolicyCenter instances connected to this workset and the edition data would remain. This is now fixed and editions are cleared from PolicyCenter if deleted using the APD cloud UI.
APD‑2522 Under some circumstances, it was possible for a codeIdentifier collision to occur when managing multiline products using APD, which prevented REST API generation. This issue has been resolved.
APD‑2557 During code generation, some references to display keys used the display name instead of the display key code, causing compilation errors when code generation was performed in locales other than en_US. Code generation now correctly uses the display key code instead of the display name.

Platform improvements and resolved issues

ID Description
ISGOSU‑435 Gosu fragment names are no longer cached in a string pool, resolving a memory leak.
ISPL‑8773 Messaging Statistics no longer throws a NullPointerException when the primary object has been deleted.

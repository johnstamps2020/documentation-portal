Querying logs

Kubernetes logs in SumoLogic contain metadata you can use for filtering. For example, you can search for cluster, namespace, deployment, pod, and more.

Querying Kubernetes logs

SumoLogic indexes Kubernetes resource logs with their native names. You can query those resources in SumoLogic. In your queries, you can use the following fields to filter logs:

  • cluster
  • namespace
  • deployment
  • pod
Note: You cannot use these fields when viewing logs in real time.

Filtering logs

In your queries, you can combine the Kubernetes fields with other parameters. This includes specific types of log messages and events. For example, you can search for log messages that contain the word "error" or "login".

The following example displays errors and exceptions for a ClaimCenter deployment in the namespace devsat:
namespace = devsat and deployment = tenant-project-test-cc-all and (exception or error)
You can also use Field Browser to display and hide fields. You can also use it to narrow down the results of your searches.

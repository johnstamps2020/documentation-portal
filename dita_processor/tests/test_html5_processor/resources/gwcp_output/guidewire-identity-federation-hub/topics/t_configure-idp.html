Configure your IdP for a browser application

Ensure that your IdP supports the needs of your application.

Your IdP must be able to provide the security information and data that your application requires.
  1. Ensure that your IdP is able to assign roles for each request. See Specify roles for browser-based applications.
  2. Track the data that is required by Guidewire Hub. This includes the following:
    • firstName
    • lastName
    • email
    • groups
  3. Track the data that is required for the data access levels that your application supports. See also Choose a data access level.
    Access level Information to track
    cc_gwabuid Your IdP must track the abuid of the contact that represents the specialist that is associated with this user. For example, an employee of Classic Auto Shop must track gwabuid = "abuid_for_classic_auto_shop.
    cc_policyNumbers Your IdP must track the policy numbers associated with each user. For example, track that the insured Ray Newton, rnewton@email.com, has access to policy number = ["54-123456"].
    cc_username Your IdP must track the username of each user as stored in ClaimCenter. For example, for adjuster Andy Applegate, track username = "aapplegate".
    Note: Users who do not have one of the above tracked will not have any resource access in ClaimCenter. These users will be able to access only generic endpoints, such as ones to acquire metadata.

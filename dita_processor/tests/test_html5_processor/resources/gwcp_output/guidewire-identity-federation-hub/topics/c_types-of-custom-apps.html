Types of custom applications

You can develop the following types of custom applications:

Browser-based application

A browser-based application typically involves a user navigating through a user interface. The user then performs some action that triggers a call to an InsuranceSuite application.

When you develop a browser-based application, you must choose the access levels that it has for accessing data from the InsuranceSuite application. You can develop applications for your internal users, policy holders, or vendors. This decision has implications on the application functionality, and also the configuration of your IdP.

The following are examples of browser-based applications:
  • A front-end application for a customer service representative to access customer data.
  • A portal for an insured to access information about their claims.
  • An application for a vendor to access information about claims they are working on.

See also

  • Designing a browser-based application

Service application

A service application typically involves no user interface. A service waits for an external system to call into it, which then triggers a call to an InsuranceSuite application. The following are examples of service applications:
  • An application that receives a request from a vendor to create a claim for a repair that the vendor made.
  • A batch system to read external data, such as from a spreadsheet file, and then create a claim for each row in it.

Specify roles for browser-based applications

The roles returned in the SAML response identify the permissions for the request.

For browser-based applications, roles are typically mapped to the groups defined in your user management system, such as Active Directory. When Guidewire Hub calls your IdP for validation, your IdP determines which groups the user belongs to, and returns it in the SAML response.
  1. Define the roles that you will use. You can base the roles on existing user groups, or create new roles.
  2. Create the YAML files that assign the permissions to the roles.
  3. Configure your IdP to identify the roles to return for each request.
  4. Prepend cc. to each role name.
  5. Include the groups in the SAML response.

Booking date for the Transaction entity

The BookingDate field on the Transaction entity represents the date and time when the transaction status was updated to Submitted. This field is used in ClaimCenter integrations with Guidewire DataHub to indicate that a transaction has been successfully sent downstream whereupon it cannot be further modified. When the BookingDate on a transaction is non-null, the transaction is recognized to be in a committed state and cannot be denied. DataHub can then safely extract the transaction.

The configuration parameter, AllowTransactionDenyWhenSubmitted, determines when transactions can be denied. In the base configuration, AllowTransactionDenyWhenSubmitted is set to false, which means a check or recovery can be denied in Submitting/Requesting status only. This requirement is necessary for DataHub integration. If AllowTransactionDenyWhenSubmitted is set to true, transactions can be denied in Submitted or Requested status as well.

Setting the Booking Date on import

When a transaction is imported using a web service API, the BookingDate field is set to the current date and time if the transaction is in a Submitted or later status. This is consistent with the import of new transactions using web service APIs and appropriate for DataHub use.

In a less common use case, if you import older transactions using a web service API, you will need to perform the additional steps below:
  • Add _ bookingDate to the TransactionDTO class and modify methods in the class to propogate the value to and from the entity.
  • Set the the BookingDate field during the call to a ClaimFinancialsAPI operation to a date consistent with other dates on the transaction.
Note: The BookingDate field is not set automatically during staging table import. Guidewire recommends that you set the BookingDate field appropriately during staging table import of financial transactions to ensure that they are included in DataHub. Refer to the Guidewire DataHub documentation for information on how to import converted, legacy data into DataHub.

Archiving links plugin

Archiving in BillingCenter provides a way to archive and purge inactive policy data from the GenericCenter database. During archiving, GenericCenter writes data about individual instances of PolicyPeriod entities and all associated data to an archive store. It then purges the associated data, but not the individual PolicyPeriod instances from the GenericCenter database.

In this process, GenericCenter archives any instances of entity types that a policy period owns. GenericCenter considers any entity type with a foreign key link to a PolicyPeriod entity instance with archivingOwner attribute set to target as being owned by that policy period instance. The foreign key can be either direct or indirect.

For instance, in the base configuration, all InvoiceItem entities have a direct foreign key to PolicyPeriod. PolicyCommision has a direct foreign key to PolicyPeriod as well. ItemCommission has an indirect foreign key to PolicyPeriod through both InvoiceItem and PolicyCommision. The archivingOwner attribute for each of these foreign keys is set to target, thus making PolicyPeriod the owner of all of these entity types. A PolicyPeriod instance also owns any entity type with a foreign key from PolicyPeriod to the entity type with the archivingOwner attribute set to source.

The PolicyPeriod entity implements the RootInfo delegate to become the root of the GenericCenter archive domain graph. To be in the archive domain graph, an entity type must implement the Extractable delegate. The set of all entities that implement the Extractable delegate constitute the archive domain graph.

There are certain entity types for which individual entity instances may, or may not, belong in the archive domain graph, however. For instance, the Charge entity represents charges on both account and policy period entities. In other words, Charge has foreign key links to both Account and to PolicyPeriod. However, Account does not implement the Extractable delegate and is never in the archive domain graph. Only the charges associated with a policy period exist in the archive domain graph.

During archiving, GenericCenter needs to archive those charges associated with a policy period only. Thus, even though there are links from Charge to both Account and PolicyPeriod, GenericCenter can safely ignore any link from Charge to Account.

In the base configuration, GenericCenter provides a default implementation of an IBCArchivingPlugin plugin that you can use to manage foreign key links between GenericCenter entities during archiving.

IBCArchivingPlugin plugin usage

It is possible to fix most, if not all, of the archive domain graph errors by making adjustments to the GenericCenter data model. Guidewire recommends that you always attempt this approach first. Whenever possible, the best approach is to fix errors in the data model before you register a problematic link in the IBCArchivingPlugin plugin code as safe to ignore.

Also, it is not possible to use IBCArchivingPlugin plugin code to register any type of foreign key link as safe for GenericCenter to ignore. In particular, you cannot use plugin code for any of the following:

  • Links that already exist in the base GenericCenter configuration.
  • Links from entity instances inside the archive domain graph to entity instances outside the graph.
  • Links from entity instances outside the archive domain graph to other entities instances outside the graph.

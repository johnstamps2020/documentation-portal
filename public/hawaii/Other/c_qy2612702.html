Multicurrency policies

In PolicyCenter 8.0, PolicyCenter provides support for multicurrency policies. Through multicurrency, PolicyCenter provides the ability to write policies that provide insurance for assets in different currencies. For example, an insurer offers a commercial property policy that can include properties in more than one country. With multicurrency, the policy values the assets in the currency of the asset location.

In PolicyCenter 8.0, PolicyCenter is always configured as a multicurrency system, even if only one currency is defined. The data model and the business logic do not change when the display is set to single currency (the default). The multicurrency user interface elements are only visible in the base configuration when you enable multicurrency display mode. Even when multicurrency display is not enabled, the currency-related fields are still populated within the data model. In single currency display mode, PolicyCenter does not display the user interface to change those fields.

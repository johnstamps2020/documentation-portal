Example implementation of method safeCrossDomainLinks

Note: Guidewire recommends that you always attempt to resolve archive domain graph errors through data model configuration before you register a foreign key link in your implementation of the IBCArchivingPlugin plugin.

Gosu class BCArchivingPlugin provides the implementation for GenericCenter archiving plugin IBCArchivingPlugin. Use the safeCrossDomainLinks method on this class to register specific types of foreign key links as safe to ignore while archiving an instance of the archive domain graph. These kinds of foreign keys link an entity instance inside the archive domain graph to another entity instance inside the graph. In this case, however, GenericCenter never archives the link target entity instance.

An example of such a relationship in the base configuration is the link from LineItem to TAccount. GenericCenter does archive ChargePaidFromAccount line items. But, GenericCenter does not archive their link target TAccount (for Account Unapplied) as the instance of TAccount in this case belongs to Account.

Archive code considers the LineItem → TAccount link as a cross-domain link. As the link target (TAccount) implements the Extractable delegate, archive code considers it possible for the TAccount instance to belong to a different PolicyPeriod graph than the LineItem instance. If not handled correctly, GenericCenter throws an error while attempting to archive the graph instance and the archive operation fails. In the base configuration, GenericCenter data model code sets this particular link as safe to ignore.

The following sample entity metadata code defines an entity that has foreign keys to two different policy periods, both of which are, obviously, in the archive domain graph.

<?xml version="1.0"?>
<entity xmlns="http://guidewire.com/datamodel" 
      entity="MyExtractableCustomEntity" 
      desc="Custom entity inside of the Archiving graph. Has FK to PolicyPeriod." 
      table="myextractablecustomentity" 
      type="retireable">
  <foreignkey columnName="PolicyPeriodID" fkentity="PolicyPeriod" name="PolicyPeriod" nullok="true"/>
  <foreignkey columnName="AnotherPolicyPeriodID"
  fkentity="PolicyPeriod" name="AnotherPolicyPeriod" nullok="true"/>
  <implementsEntity name="Extractable"/>
</entity>

Custom entity MyExtractableCustomEntity defines two link targets, each of which is an instance of PolicyPeriod, the root entity type in the archive domain graph. In this case, the archive code fails with a runtime error while attempting to traverse the set of entity instances in the archive domain graph during an archive operation. To remove this runtime error, you need to register the link in IBCArchivingPlugin code as safe to ignore.

The following sample code for method BCArchivingPlugin.safeCrossDomainLinks registers the link from MyCustomExtractableEntity to AnotherPolicyPeriod as safe for GenericCenter to ignore.

class BCArchivingPlugin implements IBCArchivingPlugin {
  override function safeCrossDomainLinks() : Set<ILinkPropertyInfo> {
    return { MyExtractableCustomEntity#AnotherPolicyPeriod.PropertyInfo as ILinkPropertyInfo }
  }
}

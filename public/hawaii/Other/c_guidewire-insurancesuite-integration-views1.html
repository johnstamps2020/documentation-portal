Guidewire InsurancePlatform Integration Views

Integration Views allows you to select and retrieve data from the GenericCenter database and further compose and transform that data in a custom manner to meet business needs. Moreover, the feature allows you to define a stable, versioned contract for this data processing. In addition, the feature provides a simple, declarative way to map underlying GenericCenter data into a serialized format that conforms to the contract.

You define the contract explicitly and independently from the underlying GenericCenter data model. In this way, the contract is decoupled from the data model. Because of this manner of definition, when you change the contract, the change is both explicit and intentional.

Integration Views allows different versions of the contract to coexist. This simultaneous existence allows you to evolve the contract and the systems that rely upon it in a controlled fashion to meet business needs.

At the same time, Guidewire can improve the GenericCenter data model. By strategically decoupling your data integration process from the Guidewire data modeling process, Integration Views allows you and Guidewire to work together without interference to achieve common progress.

Integration Views allow you to select data from the GenericCenter database and further compose and transform that data in a custom manner to meet business needs. The custom manner in which you integrate the GenericCenter data with other data forms a stable external contract. With the contract, you define the custom processes for publishing and retrieving data. Moreover, you can effectively and efficiently convert data from multiple sources into business information.

The component GenericCenter data you select during the integration process comes by following the contract. The contract is implemented by means of a stable external interface. This interface connects your system to the GenericCenter database but is independent of and separate from it. In this way, your data integration and conversion process is decoupled from the GenericCenter database.

As a result of this decoupling, you have the freedom to alter and evolve the external contract that defines your integration process in accordance with your business needs. You can exercise this freedom without fear of contaminating or destroying the GenericCenter data model. In a similar vein, the decoupling allows Guidewire the freedom to improve the GenericCenter data model. The decoupling allows the improved model to accommodate more of your business needs more effectively. By decoupling your integration process from the Guidewire data modeling process, you and Guidewire can work together but separately to achieve progress.

Integration Views allow you to define a stable, versioned contract for data extracted or retrieved from PolicyCenter, along with a simple, declarative way to map the underlying PolicyCenter data into a serialized format that conforms to that contract. The contract is defined explicitly and decoupled from the underlying PolicyCenter data model, ensuring that the contract is only changed explicitly and intentionally, and multiple different versions of the contract can simultaneously co-exist, allowing you to manage the evolution of those contracts and the systems that consume them in a controlled fashion.

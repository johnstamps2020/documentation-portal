Metadata

You must add metadata for all elements in the product model tree in fields labeled (*PD). These fields are required for product generation.

The metadata includes business descriptions and codes, such as:
  • General product details
  • Line details and risk objects
    • Fields
    • Coverages, conditions, and exclusions
    • Drop-down lists associated with fields, coverages, conditions, and exclusions

The product model is not valid until all metadata has been entered.

How to specify metadata

There are several ways to specify metadata for the product model:
  • Let APD create default values when importing or updating a mind map.

    When importing an XMind mind map, APD generates values for (*PD) metadata fields by default. This setting is on the Load Product Template popup, where Default additional fields? is set to Yes by default. Default values are provided only for (*PD) fields that are empty.

  • Enter (*PD) values in APD screens directly.
  • To provide your own values for Short Name (*PD) fields, you can define short names in the mind map.

Short Name (*PD)

The Product Definition screens have Short Name (*PD) fields that must be filled in before generating the product.

When importing the mind map, APD generates values for the Short Name (*PD) field by default. With the exception of the Product and Line topics, short names are generated for product model elements (all topics with markers). In most cases, you can use this generated short name.

If APD encounters a problem with the generated short name, then it displays a warning. This can happen if the generated name is too long or contains characters that are not allowed. For these cases, you can define the short name in the mind map, then reimport the mind map.

Another reason for specifying the short name in the mind map is to have more control over the short name. For example, in the short name you can substitute abbreviations for words.

For products created directly in APD, you must enter values for all Short Name (*PD) fields.

Mind map label for short name

For all topics with markers (except Product and Line) in the XMind mind map, you can specify the short name in a label. The label becomes the Short Name (*PD) on APD screens.

Short name requirements

Short names must meet the following requirements:
  • Must be unique within the product line
  • Maximum of 45 characters, unless noted below
  • No spaces or special characters
  • Clause terms have a maximum of 15 characters

When importing a mind map, APD displays a warning if it encounters labels for short name that do not meet these requirements.

Add metadata

About this task

Note:

Do not add prefixes to codes that identify the line or risk object. The build process will do this for you. These codes end with excluding prefix (*PD).

Procedure

  1. In your visualized product, navigate to the Product Design screen.
  2. Starting with the Details tab, enter values for all fields appended with (*PD).
  • Fields
    1. On the Fields tab, enter description and name.
      • Description (*PD) – Describe the business use of this field.
      • Name (*PD) – Add the field code name using conventions defined for the project.
      • Category – Do not enter anything. This field is intended for future use.

      If a field is a drop-down list, see Add metadata for drop-down list (shared topic).

  • Coverages, Exclusions, and Conditions
    1. On the Coverages, Exclusions, or Conditions tab, enter Description (*PD) and Code, excluding XXX prefix (*PD) for each row in the table.
    2. For each row in the table, click Terms to update description and code for each term.
    3. For each drop-down list, define metadata. See Add metadata for drop-down list (shared topic).
    4. At the top of the tab, click Manage Clause Categories to update clause category descriptions and codes.
  • Risk objects
    1. Click the Children tab to set metadata on risk objects.
      A table contains a link to each risk object.
    2. For each risk object, click the link to display the Risk Object Definition popup.
    3. On the Details tab, enter metadata.
      OptionDescription
      Description (*PD) Describe the risk in business terms
      Menu Label for Generated UI (*PD)  
      Risk Object Entity Type Name, excluding prefix (*PD)
    4. Click the Fields tab to enter metadata for fields on the risk object. Use the guidance provided above for completing the Fields metadata.
    5. Continue this process for all tabs and all parts of the product model.

    Add metadata for drop-down list

    On the Fields tab, you must provide metadata for each item in a Drop-down list.

    Procedure

    1. On the Fields tab, click on Definition for a drop-down list.
      The Drop-Down Definition popup appears.
    2. In Typelist (*PD), give the typelist a meaningful name.
    3. For each typelist value, enter Label, Description (*PD) and Short Name (*PD).

    Adding metadata for coverages, exclusions, and conditions

    On the Coverages, Exclusions, and Conditions tabs, enter metadata for all fields on these clauses. For each clause in the product model, you must define:

    • Clause categories – Click Manage Clause Categories to define these categories.
    • Description (*PD) – Enter description using business terms.
    • Code, excluding XXX prefix (*PD) – Follow Guidewire or project protocols.
    • Clause terms, including clause schedule terms – Click on Terms to add metadata for the term.
    • Definition of the clause – Click Details to display the Definition popup for the clause.

New and changed in GenericCenter 9.0.2

For resolved and known issues in 9.0.2, see the Release Notes.

New in GenericCenter 9.0.2

Commission flat amount overrides

In addition to commission rate overrides, BillingCenter now supports flat amount overrides. You can create flat amount overrides through code. The base configuration does not include a way to create flat amount overrides through the user interface.

As of this release, there are three approaches for determining the commission amount for a given charge: commission plan rate, commission rate override, or commission flat amount override. Regardless of the approach used, BillingCenter still divides the commission amount into a set of item commission objects associated with the appropriate invoice items.

To support flat amount overrides, the ChargeCommissionRateOverride entity now includes an Amount field as well as a Rate field.

  • To specify a rate override, provide a value in the Role and Rate fields. Set the Amount field to null.
  • To specify a flat amount override, provide a value in the Role and Amount fields. Set the Rate field to null.

BillingCenter throws an exception if you attempt to create a ChargeCommissionRateOverride where both Rate and Amount are null or both Rate and Amount are non-null.

The following methods have been added to support flat amount commissions:

ChargeInitializer

overrideCommissionAmount( PolicyRole, MonetaryAmount ) : void

  • Overrides the commission amount for the given role to the given monetary amount.

Charge

overrideCommissionAmount( PolicyRole, MonetaryAmount ) : void

  • Overrides the commission amount for the given role to the given monetary amount.

getChargeLevelCommissionAmountOverride( PolicyRole ) : MonetaryAmount

  • Returns the flat amount override, or NULL if there is a rate override or if no override exists.

getTypeOfOverrideIfAny( PolicyRole ) : ChargeCommissionOverrideType

  • Returns the type of override (Rate or Amount), or NULL if no override exists.

ChargeCommission

getCommissionAmount() : MonetaryAmount

  • Returns the flat amount commission, if one exists. Otherwise, this returns the calculated commission amount based on the rate.

Rule comparison

GenericCenter now includes a rule comparison feature for business rules, which enables the side-by-side comparison of existing rules and rules to be imported. The Compare Rules screen highlights changes and helps resolve rule conflicts.

GenericCenter now includes a rule comparison feature for underwriting rules, which enables the side-by-side comparison of existing rules and rules to be imported. The Compare Rules screen highlights changes and helps resolve rule conflicts.

Changed in GenericCenter 9.0.2

Deprecated ICommission plugin interface methods

The ICommission plugin uses the ICommission interface. Methods in this interface that are deprecated are listed below as well as appropriate replacement methods. Update configuration code to employ the replacement methods.

Deprecated method: getCommissionRate()

Replace calls to the deprecated getCommissionRate method with references to the new enhancement property ChargeInitializer.overrideCommissionRate.

Purging claims with bulk invoices

It is now possible to purge claims even if they are associated with bulk invoices. However, there are some restrictions. See also Understanding claim purging.

Common tasks

Add attributes

In visualize mode, you can configure additional modifiers and attributes for a product line.

Before you begin

Product Design Mode must be set to Designer or above.

About this task

The following example defines a new attribute, monetary amount, to a product line, such as Small Business Product. In this example, the monetary amount captures information, such as the average turnover for the last three years.

Procedure

  1. Start a submission for the visualized product.
  2. In the left sidebar, select Risk Details.
  3. In the Risk Details screen, select Add.
  4. Enter a descriptive Label, such as Average turnover for the last three years.
  5. Select a data type for the new attribute, in this case Monetary amount.

Add new clause categories

In visualize mode, you can add coverage categories, coverages and terms for a product line. These coverage categories, coverages and terms apply at the policy level. For example, you can add a new Liability Coverage to the policy.

About this task

The process of defining new Coverage Categories, Coverages and Terms is described below for the Risk Object. The process is identical for defining Coverage Categories, Coverages and Terms on the Policy Line level. The only difference is the starting point. You can define Coverage Categories, Coverages and Terms for the Policy Line from the Product Line tab.

These steps add a coverage category.

Procedure

  1. Start a submission for the visualized product.
  2. In the left sidebar, select Product Line > Risk Details.
  3. In the Risk Details screen, select Add Primary Cover Set > Itemised Cover.
  4. On the Add New Category screen, enter information for the new coverage category.
  5. Click OK to save the new coverage information in your product line.

Add categories on the Product Definition screen

You can add new categories and edit and manage existing categories on the Product Definition screen.

Procedure

  1. On the Product Definition screen, go to the Coverages, Exclusions, or Conditions tab.
  2. In the Category column, select Add New Category.
  3. To make bulk changes, select the Manage Clause Categories button.

Add or edit conditions and exclusions

You can define conditions, exclusions, and exclusion categories for a coverage offered at the policy or risk object level.

Procedure

  1. Start a submission for a visualized product.
  2. In the left sidebar, select Risk Details.
  3. In the Risk Details screen, select the Conditions and Exclusions Library tab and click Add Conditions and Exclusions.
  4. In the Search Exclusions and Conditions for product screen, you can add conditions and exclusions. You can also search for and edit an existing exclusion or condition.
  5. Select Add Selected Exclusions and Conditions to save your work.

Modify risk object coverage terms

In the Product Definition screen, you can modify the terms for a coverage on a risk object.

About this task

The following example shows the terms for the Business Interruption coverage for the Small Business product line.

Procedure

  1. On the Product Definition screen, select the Coverages tab.
  2. For a coverage, such as Business Interruption, click Terms.

    The resulting Clause Terms screen displays two tables: Clause Terms and Clause Schedule Terms for scheduled items. You can define both Clause Terms and Clause Schedule Terms.

Managing terms associated with coverages, exclusions, and conditions

For each clause (coverage, exclusion, or condition) that has a term, you must complete the definition of the term. For each clause, you can define both Clause Terms and Clause Schedule Terms.

Define drop-down lists

You can define the different types of drop-down lists in the mind map. You can also create new types of drop-down lists in APD screens. Drop-down lists become typelists in GenericCenter.

About this task

If more than one field uses the same list, you only need to define it once. You then point all the other fields to the one where you have defined the list of entries. This field is the Owning Attribute of shared code lists.

These instructions define a new drop-down list called CarType with values Sedan and Coupe.

Procedure

  1. On the Manual tab, select a product, such as Small Business.
  2. Click Risk Details in the left sidebar. GenericCenter displays the Risk Details screen, LineRisk Details tab.
  3. In the line, find or create a product part that is a drop-down list type. For example, in Small Business, go to the Risk Details > SB LineRisk Details screen.
  4. Click Add to add a new product part.
  5. In the new row, enter a Label, such as “What type of car do you drive?”
  6. Select Drop-down list as the Type.
  7. Next to the Value entry, select Add New Choice from the green down arrow.
  8. If you selected Add New Choice, enter the name and description of a choice selection, such as Sedan. Enter a product code, such as SDN. Use typelists defined in GenericCenter as an example. You can view typelists in Studio in the configuration/config/Extensions/Typelist folder.
  9. Next to the Value entry, select Edit Definition from the green down arrow.
  10. Click Add to add another choice. For example, Coupe with a code of CPE.
  11. In the Typelist (*PD) field, enter the name of the typelist. Use typelists defined in GenericCenter as an example. For example, name this typelist CarType.

Example

The Drop-Down Definition popup has the following fields:

Field Description
Owning Attribute of shared code lists If more than one field uses the same typelist, you only need to define it once. To use an existing typelist, select that typelist in the drop-down list.
Typelist (*PD) Name of the typelist.
Do NOT re-build the Typelist

Select Don’t Do It if you do not want the rebuild the typelist when you generate the product.

If a typelist is flagged to not regenerate, its codes are not saved and the codes you create on APD screens are not used to maintain the typelist. It is assumed that these are typelists that are maintained centrally and are not owned or maintained by the product. For example, typelists such as country codes that are defined in the base configuration.

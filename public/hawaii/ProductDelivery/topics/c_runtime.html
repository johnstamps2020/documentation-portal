Finalizing and generating the product

When you have refined and extended the visualized product definition created during visualization, you are ready to finalize the product. Finalization is the phase where the visualized product is used to generate the artifacts needed for PolicyCenter including entities, product clauses, and user interface elements.

Finalization has multiple steps:

  1. Product Design Mode must be set to Developer.
  2. At least one coverage must have dummy rating This is so that the finalize process creates a cost element.
  3. All metadata codes and descriptions must follow either Guidewire suggested protocols, or protocols agreed upon by your the project.
  4. Review all content.
  5. Validate the product definition.
  6. Generate the product code to create the installed product.

After finalization, you can view installed products in the Automated tab of the New Submissions screen in PolicyCenter. You can manage these products in the same way as products defined in the base configuration.

You complete the product development using the development tools provided by PolicyCenter. These tools are described in the PolicyCenter documentation. You can customize the user interface, add integrations with other applications, and make other changes to meet your needs.

Visualization items excluded from code generation

Some nodes and items that can be used in the XMind mind maps for visualization are not used in PolicyCenter for product code generation.

These fields include:

  • Scheduled Items
  • Basic Pricing
  • Loss/Peril data

Save frequently

Save frequently during finalization. Use Export to Template on your model prior to the final build. Remember, if you drop your database before the final build is complete, you will lose your work on the product model in APD

After generating product code

After generating the product code, the next steps are additional configuration in PolicyCenter to manage rules, rating, availability, underwriting, forms, integrations, among others using tools such as Product Designer and Guidewire Studio.

Modifying the product definition after code generation

After completing product code generation, deletions in the mind map or XML template file are not supported and are not reflected on reimporting the files. You must make these deletions manually in Guidewire Studio.
